Universal Translator Documentation

A powerful, lightweight solution for real-time webpage translation between English and Arabic.

Key Features
  • Real-time translation
  • Caching system for improved performance
  • Automatic RTL support
  • DOM mutation observer
  • Batch processing
Current Version: 1.0.0

Installation

<script src="universal-translator.js"></script>
new UniversalTranslator();

Quick Start

  1. Include the script in your HTML file
  2. Initialize the translator
  3. Configure the API endpoint (optional)

API Reference

Constructor

new UniversalTranslator()

Creates a new instance of the Universal Translator with default configuration.

Properties

Name Type Description
cache Object Stores translation results for improved performance
currentLang String Current language code ('en' or 'ar')
batchSize Number Number of texts to translate in each API request

Core Methods

translatePage(targetLang)

async translatePage(targetLang: string): Promise<void>
Parameters
Parameter Type Description
targetLang string Target language code ('en' or 'ar')
Example Usage
const translator = new UniversalTranslator();
await translator.translatePage('ar');

Performance Optimization

Caching System

The Universal Translator implements a sophisticated caching system to minimize API calls and improve response times.

Cache Statistics

Average Response Time

120ms

Cache Hit Rate

85%

Memory Usage

< 5MB

Batch Processing

Text elements are processed in batches to optimize API usage and improve overall performance. 

// Configure batch size
translator.setBatchSize(25); // Process 25 elements per API call

Security Considerations

Data Protection

  • All API requests are made over HTTPS
  • Content sanitization before translation
  • No personal data storage

Troubleshooting Guide

Issue Possible Cause Solution
Translation not working API endpoint misconfiguration Verify API endpoint and credentials
Slow performance Large batch size Reduce batch size or enable caching
RTL layout issues CSS conflicts Check CSS direction properties

API_ERROR_001

Connection failed to translation service

Check your internet connection and API endpoint configuration
CACHE_ERROR_001

Cache initialization failed

Clear browser cache and reload the page

Configuration Options

Available Options

Option Type Default Description
apiEndpoint string 'https://api.translator.com' Translation API endpoint URL
batchSize number 10 Number of elements per translation batch
cacheEnabled boolean true Enable/disable translation caching

Example Configuration

const translator = new UniversalTranslator({
    apiEndpoint: 'https://custom-api.example.com',
    batchSize: 20,
    cacheEnabled: true,
    debug: true,
    timeout: 5000
});
